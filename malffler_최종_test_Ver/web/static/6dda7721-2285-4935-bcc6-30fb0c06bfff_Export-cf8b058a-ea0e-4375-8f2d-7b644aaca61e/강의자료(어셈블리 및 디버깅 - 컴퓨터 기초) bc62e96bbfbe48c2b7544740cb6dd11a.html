강의자료(어셈블리 및 디버깅 - 컴퓨터 기초)

1. 컴퓨터 기초

기본 데이터 유형 :

8비트 묶음은 바이트(Byte)를 구성한다.

단일 바이트는 16진수 2개로 나타내고, 16진수 하나의 크기는 4바이트이다.

바이트 외에 다른 데이터 유형이 있다.

워드(Word)의 크기는 2Byte(16Bit), 더블워드(Dword)의 크기는 4Byte(32Bit), 쿼드워드(Qword)는 8Byte(64Bit)이다.

1.1 메모리

메인 메모리(RAM)는 코드와 데이터를 저장한다.

첫 번째 주소는 0에서 시작하고 마지막 주소는 사용하는 하드웨어와 소프트웨어에 따라 다르다.

주소와 값은 16진수로 표현한다.

1.2 CPU

CPU(Central Processing Unit, 중앙처리유닛)은 명령어(또는 기계 명령어)를 실행한다.

CPU가 실행하는 명령어는 바이트 순서로 메모리에 저장되어 있다.

명령어를 실행하는 동안 필요한 데이터(바이트 순서로 저장)는 메모리에서 가져온다.

CPU 자체적으로 칩 내에 레지스터 집합(register set)이라는 작은 메모리 집합을 가지고 있다.

이때 레지스터는 실행 중인 메모리에서 가져온 값을 저장하는 데 사용된다.

1.3 프로그램 기초

컴파일 과정

  1. C 또는 C++ 같은 고급언어로 소스코드 작성
  1. 프로그램의 소스코드는 컴파일러를 통해 실행
    컴파일러는 고급언어를 목적파일(Object File / Machine code)이라 불리는 중간 형식으로 변환
  1. 목적 파일은 링커에 전달된다.
    링커는 목적 파일을 필요 라이브러리(DLL)과 연결하여 시스템에서 실행 가능한 실행 파일로 만든다.

1.3.1 메모리에 프로그램 로드

디스크 상의 실행파일을 더블클릭하면 프로세스 메모리를 운영 시스템에서 할당하고, 운영 시스템이 할당한 메모리로 실행 파일을 로드한다.

  • 힙 : 프로그램을 실행하는 동안 동적 메모리 할당을 위해 사용하고, 그 내용들은 다양할 수 있다.
  • 스택 : 지역변수, 함수 인수, 반환 주소를 저장하는데 사용한다.

메모리 로드된 프로그램 코드에서 입력을 받거나 출력을 하기 위한 코드가 있을 경우 CPU는 입력/출력 장치와 상호 작동한다.

프로그램 실행 시 수행 단계

  1. 프로그램(코드와 데이터 포함)을 메모리에 로드한다.
  1. CPU는 명령어를 가져온 후 디코딩하고 실행한다.
  1. CPU는 메모리에서 필요한 데이터를 가져온다.
    데이터 역시 메모리에 기록될 수 있다.
  1. CPU는 필요한 경우 입력/출력 시스템과 상호작용을 할 수 있다.
